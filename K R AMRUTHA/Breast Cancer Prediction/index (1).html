Algorithm Overview

Logistic Regression

A probabilistic linear classifier that models the log-odds of a binary outcome. Particularly effective when features are linearly separable and interpretability is required.

Tuned with GridSearchCV L1/L2 Regularisation cv=10

K-Nearest Neighbours

An instance-based lazy learner that classifies samples based on majority vote from K closest training examples. Performance is sensitive to feature scale and K selection.

K ∈ [1, 20] Euclidean / Manhattan Distance Weighting

STEP 07

Logistic Regression — Tuned

lr = LogisticRegression(max_iter=10000)

param_lr = {
    'C':       [0.001, 0.01, 0.1, 1, 10],

    'solver':  ['liblinear']

  • GridSearch finds the best regularisation strength C and penalty

// 05 — best practices

Best Practices to Reach 99% Accuracy

TIP 01
Mandatory Scaling — Always apply StandardScaler before fitting KNN or Logistic Regression. Distance-based and gradient-descent methods are sensitive to feature magnitude.
TIP 02
Strong Cross-Validation — Use cv=10 in GridSearchCV. 10-fold CV provides stable performance estimates and reduces variance in hyperparameter selection.
TIP 03
Tune Logistic Regression Properly — Search C ∈ [0.001 … 100] with both L1 and L2 penalties using the liblinear solver for reliable convergence.
TIP 04
Tune KNN Carefully — Search K ∈ [1, 20] with both uniform and distance weights, and compare Euclidean vs Manhattan distance metrics.
TIP 05
Feature Selection — Apply SelectKBest(f_classif, k=5) to keep only the top ANOVA-ranked features. Removing noise can push accuracy from ~96% to ~99%.
