04 — Feature Engineering

Data Preprocessing

Raw data requires transformation before it can be used in ML models. We apply StandardScaler to numerical columns (zero mean, unit variance) and OneHotEncoder to categorical columns. A ColumnTransformer orchestrates both transformers in a single pipeline step.

python 
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.compose    import ColumnTransformer

# Separate features and target variable
X = df.drop('label', axis=1)
y = df['label']

# Identify column types
categorical_cols = X.select_dtypes(include=['object']).columns.tolist()
numerical_cols   = X.select_dtypes(exclude=['object']).columns.tolist()

print("Categorical columns:", categorical_cols)
print("Numerical  columns :", numerical_cols)

# Build ColumnTransformer pipeline
preprocessor = ColumnTransformer(transformers=[
    ('num', StandardScaler(),                       numerical_cols),
    ('cat', OneHotEncoder(handle_unknown='ignore'), categorical_cols)
])

# Fit and transform feature matrix
X_processed = preprocessor.fit_transform(X)
print("\nProcessed feature shape:", X_processed.shape)
StandardScaler ensures all numerical features contribute equally to distance calculations in KNN. OneHotEncoder converts categorical labels into binary vectors so the model can process them mathematically without introducing ordinal bias.

06 — Custom KNN Implementation

KNN Model — Generated Code

A custom K-Nearest Neighbours classifier is implemented from scratch using Euclidean distance. The implementation covers distance calculation, neighbour retrieval, and majority-vote prediction.

python 
import numpy as np
from collections import Counter
from sklearn.metrics import accuracy_score, classification_report, confusion_matrix

# ── Step 1: Euclidean Distance ────────────────────────────
def euclidean_distance(a, b):
    return np.sqrt(np.sum((a - b) ** 2))

# ── Step 2: Get K Nearest Neighbours ─────────────────────
def get_neighbors(X_train, y_train, test_point, k):
    distances = []
    for i in range(len(X_train)):
        dist = euclidean_distance(X_train[i], test_point)
        distances.append((dist, y_train.iloc[i]))
    distances.sort(key=lambda x: x[0])
    neighbors = [distances[i][1] for i in range(k)]
    return neighbors

# ── Step 3: Majority Vote Classification ─────────────────
def predict_classification(neighbors):
    vote_counts = Counter(neighbors)
    return vote_counts.most_common(1)[0][0]

# ── Step 4: Full KNN Predict Pipeline ────────────────────
def knn_predict(X_train, y_train, X_test, k):
    predictions = []
    X_train_arr = X_train.toarray() if hasattr(X_train, 'toarray') else np.array(X_train)
    X_test_arr  = X_test.toarray()  if hasattr(X_test,  'toarray') else np.array(X_test)
    for test_point in X_test_arr:
        neighbors  = get_neighbors(X_train_arr, y_train, test_point, k)
        prediction = predict_classification(neighbors)
        predictions.append(prediction)
    return predictions

# ── Step 5: Evaluate for k=5 and k=3 ─────────────────────
for k in [5, 3]:
    print(f"\n--- Evaluating Custom KNN with k={k} ---")
    y_pred = knn_predict(X_train, y_train, X_test, k)
    acc    = accuracy_score(y_test, y_pred)
    print(f"Accuracy: {acc:.4f}")
    print("\nClassification Report:")
    print(classification_report(y_test, y_pred))
    print("Confusion Matrix:")
    print(confusion_matrix(y_test, y_pred))

10 — Concepts Explained

Code Explanation

A concise, student-friendly breakdown of the algorithms and evaluation metrics used in this project.

🔵
KNN Algorithm
KNN classifies a new data point by looking at the K closest training examples and taking a majority vote. Closeness is measured using Euclidean distance. Smaller K captures local patterns but is sensitive to noise; larger K smooths the boundary and generalises better.
📈
Logistic Regression
Logistic Regression estimates the probability of a class using the sigmoid function applied to a weighted sum of features. It is fast, interpretable, and works best when data is linearly separable. Weights are learned via gradient-based optimisation.
🎯
Accuracy
Accuracy = (Correct Predictions) / (Total Predictions). It tells us the overall percentage of correctly classified samples. It can be misleading on imbalanced datasets, so it is always reviewed alongside precision, recall, and F1-score.
🔍
Precision
Precision = TP / (TP + FP). Of all instances the model labelled as "attack", how many were genuine attacks? High precision minimises false alarms — critical in security systems to avoid wasting analyst resources on benign traffic.
📡
Recall
Recall = TP / (TP + FN). Of all actual attacks in the dataset, how many did the model successfully detect? In cyber security, recall is often prioritised over precision because missing a real attack can be catastrophic.
⚖️
F1-Score
F1-Score = 2 × (Precision × Recall) / (Precision + Recall). The harmonic mean of precision and recall. Useful as a single summary metric that penalises extreme imbalances between false positives and false negatives.

12 — Conclusions

Final Inference

Based on the experimental results, the following conclusions are drawn regarding model suitability for cyber intrusion detection.

  • 🏆
    KNN outperformed Logistic Regression across all metrics
    KNN (k=5) achieved 80.55% accuracy versus Logistic Regression's 75.10%. The non-parametric, distance-based nature of KNN is better suited to capturing complex, non-linear decision boundaries inherent in network intrusion data.
  • ⚙️
    K value significantly affects model performance
    Increasing K from 3 to 5 improved accuracy from 78.40% to 80.55%. A larger K smooths the decision boundary, reduces sensitivity to outlier points, and leads to more stable and reliable predictions on unseen traffic samples.
  • 🛡️
    KNN is effective for cyber intrusion detection tasks
    The instance-based memory of KNN allows it to identify unusual network patterns that deviate from known normal behaviour. This makes it a strong candidate for anomaly detection tasks in network security where reference patterns are well-defined.
  • ⚡
    Logistic Regression provides faster inference but lower accuracy
    While Logistic Regression trains and predicts substantially faster due to its linear formulation, it lacks the flexibility to model complex feature interactions. It is more suitable for real-time or resource-constrained deployments where speed and interpretability are prioritised over raw classification accuracy.
