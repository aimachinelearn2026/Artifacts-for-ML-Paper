SQL Injection Detection using Machine Learning

🔹 Problem (Prompt)

Web applications are highly vulnerable to SQL injection attacks, where attackers manipulate database queries to gain unauthorized access. These attacks can lead to data breaches, data loss, and system compromise. Detecting such attacks manually is difficult and error-prone, which creates a need for automated and intelligent detection systems.

🔹 About the Project

SQL Injection is one of the most dangerous vulnerabilities in web applications, listed in the OWASP Top 10. It occurs when an attacker inserts malicious SQL queries into input fields, allowing them to access or manipulate sensitive database information.

Traditional security mechanisms often rely on predefined rules, which are not effective against evolving attack patterns. Therefore, this project explores the use of Machine Learning techniques to automatically detect SQL injection attacks.

In this project, we treat SQL injection detection as a classification problem, where input queries are classified as either Normal or Malicious. Two machine learning algorithms are used:

  • K-Nearest Neighbors (KNN): Classifies queries based on similarity using distance measures.
  • Decision Tree: Uses rule-based splitting to classify queries.

The objective of this project is to build an intelligent system that can automatically identify SQL injection attacks and improve the security of web applications.

🔹 Dataset

The dataset consists of sample SQL queries labeled as either normal or malicious. These queries include real-world inspired attack patterns such as login bypass attempts and database manipulation commands.

  • hello123 → Normal
  • ' OR 1=1 -- → Attack
  • SELECT * FROM users → Attack
  • DROP TABLE users → Attack
  • normal input → Normal

This dataset helps the model learn patterns of malicious behavior and classify new queries accordingly.

  • hello123 → Normal
  • ' OR 1=1 -- → Attack
  • SELECT * FROM users → Attack
  • normal input → Normal

🔹 Code

import pandas as pd
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.model_selection import train_test_split
from sklearn.neighbors import KNeighborsClassifier
from sklearn.tree import DecisionTreeClassifier
from sklearn.metrics import accuracy_score

# Dataset
data = {
    "query": [
        "SELECT * FROM users",
        "admin123",
        "' OR 1=1 --",
        "DROP TABLE users",
        "hello world",
        "SELECT password FROM users",
        "' OR 'a'='a",
        "normal input"
    ],
    "label": [1, 0, 1, 1, 0, 1, 1, 0]
}

df = pd.DataFrame(data)

# Convert text to numbers
vectorizer = TfidfVectorizer()
X = vectorizer.fit_transform(df["query"])
y = df["label"]

# Split
X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.2)

# KNN
knn = KNeighborsClassifier(n_neighbors=3)
knn.fit(X_train, y_train)
knn_pred = knn.predict(X_test)

# Decision Tree
dt = DecisionTreeClassifier()
dt.fit(X_train, y_train)
dt_pred = dt.predict(X_test)

# Accuracy
print("KNN Accuracy:", accuracy_score(y_test, knn_pred))
print("DT Accuracy:", accuracy_score(y_test, dt_pred))

🔹 Inference

The results show that both KNN and Decision Tree models achieved high accuracy in detecting SQL injection attacks. This indicates that machine learning can effectively classify SQL queries based on learned patterns.

However, the high accuracy is influenced by the small dataset used in this project. In real-world applications, larger and more complex datasets are required to improve generalization and robustness of the model.
