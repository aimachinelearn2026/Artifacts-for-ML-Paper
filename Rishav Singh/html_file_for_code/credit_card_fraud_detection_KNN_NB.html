📝 Paper Summary¶

The paper studies credit card fraud detection – the problem of automatically identifying whether a transaction is real (legitimate) or fake (fraudulent). It's a finance domain problem.

Why is this hard?
Out of 284,807 transactions, only 492 are fraud (0.17%). The data is heavily imbalanced — like finding a needle in a haystack.

What algorithms did the paper use?

  • KNN (K-Nearest Neighbors): Classifies a transaction by looking at its K closest neighbors. If most neighbors are fraud → it's fraud. Simple and intuitive.
  • Naive Bayes: Uses probability (Bayes theorem) to predict fraud. It assumes all features are independent of each other (that's the 'Naive' part). Very fast.

Dataset columns:

  • V1 to V28: These are PCA-transformed (anonymized) features — original features are hidden for privacy
  • Time: Seconds elapsed since first transaction
  • Amount: Transaction amount in Euros
  • Class: 0 = Legitimate, 1 = Fraud

Key findings from the paper:

  • Naive Bayes was faster to train but had lower precision (more false alarms)
  • KNN was more accurate in classifying fraud correctly but slower
  • Both models together showed complementary strengths

# ============================================================
# CELL 7a: Print Metrics for Both Models
# ============================================================

def evaluate_model(name, y_test, y_pred, y_prob):
    acc  = accuracy_score(y_test, y_pred)
    prec = precision_score(y_test, y_pred)
    rec  = recall_score(y_test, y_pred)
    f1   = f1_score(y_test, y_pred)
    roc  = roc_auc_score(y_test, y_prob)

    print(f"\n{'='*45}")
    print(f" Model: {name}")
    print(f"{'='*45}")
    print(f"  Accuracy  : {acc*100:.2f}%")
    print(f"  Precision : {prec*100:.2f}%")
    print(f"  Recall    : {rec*100:.2f}%")
    print(f"  F1 Score  : {f1*100:.2f}%")
    print(f"  ROC-AUC   : {roc:.4f}")
    return {'Model': name, 'Accuracy': acc, 'Precision': prec, 'Recall': rec, 'F1': f1, 'ROC-AUC': roc}

knn_results = evaluate_model("KNN (K=5)", y_test, y_pred_knn, y_prob_knn)
nb_results = evaluate_model("Naive Bayes (GaussianNB)", y_test, y_pred_nb, y_prob_nb)

# ============================================================
# CELL 9: Finding Best K for KNN
# ============================================================

k_values = range(1, 21)
k_accuracies = []
k_recalls = []

for k in k_values:
    model = KNeighborsClassifier(n_neighbors=k)
    model.fit(X_train, y_train)
    y_pred = model.predict(X_test)
    k_accuracies.append(accuracy_score(y_test, y_pred))
    k_recalls.append(recall_score(y_test, y_pred))

fig, axes = plt.subplots(1, 2, figsize=(14, 5))
axes[0].plot(k_values, k_accuracies, marker='o', color='steelblue')
axes[0].set_title('KNN: K vs Accuracy')
axes[1].plot(k_values, k_recalls, marker='s', color='tomato')
axes[1].set_title('KNN: K vs Recall')
plt.show()

INFERENCE: Hyperparameter tuning shows that the best K value usually falls between 3 and 9. While higher K values smooth noise, lower K values may capture specific fraud patterns more accurately. We prioritize Recall when choosing K.
