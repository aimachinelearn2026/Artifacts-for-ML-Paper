Linear Regression

🐍 In [4] — Model 1: telecommuting → has_questions INPUT 
from sklearn.linear_model import LinearRegression
from sklearn.metrics import mean_absolute_error, mean_squared_error, r2_score

X_feature = df[['telecommuting']]
y_target   = df['has_questions']

X_train, X_test, y_train, y_test = train_test_split(
    X_feature, y_target, test_size=0.2, random_state=42)

model = LinearRegression()
model.fit(X_train, y_train)
y_pred = model.predict(X_test)

mae  = mean_absolute_error(y_test, y_pred)
mse  = mean_squared_error(y_test, y_pred)
rmse = np.sqrt(mse)
r2   = r2_score(y_test, y_pred)

print(f"Coefficient (telecommuting): {model.coef_[0]:.2f}")
print(f"Intercept:                   {model.intercept_:.2f}")
print(f"MAE:  {mae:.2f}")
print(f"MSE:  {mse:.2f}")
print(f"RMSE: {rmse:.2f}")
print(f"R²:   {r2:.2f}")

💡 Inference — Linear Regression

Linear Regression is inappropriate for binary classification. An R² ≈ 0 confirms no meaningful linear relationship exists between these binary inputs and the target. The model outputs continuous values outside [0,1], which has no interpretable meaning for a fraud label. Logistic Regression is the correct choice for binary outcomes.

\( \hat{y} = \beta_0 + \beta_1 x_1 \)  |  For binary classification: use \( P(y=1|x) = \sigma(\mathbf{w}^T\mathbf{x}) \) instead

Logistic Regression

from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score, confusion_matrix, classification_report

numerical_features = ['job_id', 'telecommuting',
                       'has_company_logo', 'has_questions']
X_numeric = df[numerical_features]

    X_numeric, y_target_clf, test_size=0.2, random_state=42)

# max_iter=1000, liblinear solver for small datasets
log_reg_model = LogisticRegression(max_iter=1000, solver='liblinear')
log_reg_model.fit(X_train_clf, y_train_clf)

conf_matrix = confusion_matrix(y_test_clf, y_pred_log_reg)
class_report= classification_report(y_test_clf, y_pred_log_reg)

print(f"Accuracy: {accuracy:.2f}")
print(f"\nConfusion Matrix:\n{conf_matrix}")
print(f"\nClassification Report:\n{class_report}")

Artificial Neural Network (ANN)

🐍 In [7] — ANN with 10 hidden neurons INPUT 
import tensorflow as tf
from tensorflow.keras.models import Sequential
from tensorflow.keras.layers import Dense
from sklearn.preprocessing import StandardScaler

numerical_features_for_ann = ['job_id', 'telecommuting',
                                'has_company_logo', 'has_questions']
X_processed_ann = df[numerical_features_for_ann]
y_target_ann    = df['fraudulent']

X_train_ann, X_test_ann, y_train_ann, y_test_ann = train_test_split(
    X_processed_ann, y_target_ann, test_size=0.2, random_state=42)

# Feature scaling — critical for ANNs
scaler = StandardScaler()
X_train_scaled = scaler.fit_transform(X_train_ann)
X_test_scaled  = scaler.transform(X_test_ann)

# Build the ANN: 1 hidden layer with 10 neurons
model = Sequential([
    Dense(10, activation='relu',
          input_shape=(X_train_scaled.shape[1],)),  # Hidden layer
    Dense(1,  activation='sigmoid')                  # Output layer
])

model.compile(optimizer='adam',
              loss='binary_crossentropy',
              metrics=['accuracy'])

history = model.fit(X_train_scaled, y_train_ann,
                    epochs=50, batch_size=32,
                    verbose=0, validation_split=0.2)

loss, accuracy_ann = model.evaluate(X_test_scaled, y_test_ann, verbose=0)

y_pred_prob_ann = model.predict(X_test_scaled)
y_pred_ann = (y_pred_prob_ann > 0.5).astype(int)

conf_matrix_ann  = confusion_matrix(y_test_ann, y_pred_ann)
class_report_ann = classification_report(y_test_ann, y_pred_ann)

print(f"ANN Accuracy: {accuracy_ann:.2f}")
print(f"\nConfusion Matrix:\n{conf_matrix_ann}")
print(f"\nClassification Report:\n{class_report_ann}")

Final Inference & Recommendations

🏆 Best Model

The Decision Tree achieved the highest accuracy (97%) and macro F1 (0.97). It naturally handles binary features and requires no feature scaling. However, the unconstrained tree is prone to overfitting — it should be regularised with max_depth or replaced with a Random Forest / Gradient Boosted Trees ensemble.

⚠ Dataset Imbalance Warning

The dataset is highly imbalanced (~93% legitimate, ~7% fraudulent). All models benefit from this imbalance (even a dummy classifier predicting class 0 always gets 93%). For real-world deployment, apply SMOTE, class_weight='balanced', or use PR-AUC instead of accuracy as the primary metric.

🚀 Improvement Roadmap

1. Use text features — job title, description, requirements contain rich signals; TF-IDF + Logistic Regression or BERT-based classifiers would significantly boost performance.
2. Handle imbalance — SMOTE oversampling or class weighting.
3. Tune hyperparameters — cross-validated grid search for Decision Tree depth, KNN's k, ANN layers/neurons.
4. Ensemble methods — Random Forest or XGBoost as strong drop-in replacements.
5. Correct Naive Bayes variant — switch to BernoulliNB for binary features.
